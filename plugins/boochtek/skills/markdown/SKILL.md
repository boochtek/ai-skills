# Markdown Skill

## Formatting Checks

When available, run `make markdownlint` or similar to check for proper Markdown formatting.

## Key Rules

- There must be a blank line before and after any list (ordered or unordered).
- There must be a blank line before and after fenced code blocks.

## Markdown Flavors

- Markdown should generally follow CommonMark
- GitHub-flavored Markdown is appropriate in most situations
- When writing slides, you may use Marp extensions
    - Separate slides with 3 dashes (`---`)

## Links

- Links to other files in the same repo are **encouraged**
    - Create an Obsidian-like interconnected knowledge base
